// Architecture

Hybrid 3-Phase Pipeline

Deterministic evidence collection feeds an AI-powered analysis engine. No data is hallucinated. No sources are fabricated.

01

Phase 1

Evidence Collection

Deterministically fetches the SEN ticket and all linked CPCLIENT tickets with full comments from JIRA. Writes each ticket as a standalone evidence file to disk.

02

Phase 1.5

Intelligent Discovery

Scans evidence for repository hints — Java packages, component names, stack traces. Discovers and registers relevant code repos on EFS automatically.

03

Phase 2

AI-Powered RCA

Claude reads evidence files and explores code repositories, then generates a structured 6-section RCA with mandatory source citations for every claim.

04

Phase 3

Label Management

Automatically transitions JIRA labels: removes VERITAS, adds VERITAS-COMPLETED and RCA-AVAILABLE. Fully closes the automation loop.

// Capabilities

Built for Truth

Every design decision serves one goal: producing RCA reports that engineering teams can trust completely.

Evidence-Driven

Every statement traces to a specific JIRA ticket or code reference. Citations are mandatory, not optional.

Zero Speculation

“Not stated in available evidence” replaces guessing. No “likely”, “probably”, or “appears to be”.

PHI-Safe

Automatically detects and redacts Protected Health Information. Patient names, DOBs, SSNs, and MRNs become [REDACTED-PHI].

File-Based Evidence

Claude explores evidence files like code — Read, Grep, Glob. Scales to any number of linked tickets without prompt limits.

Code-Aware

Discovers relevant repos from evidence hints, explores source code, and cites specific files and line numbers in the RCA.

Fully Automated

Label trigger → evidence collection → repo discovery → RCA generation → JIRA update. No human in the loop.

// Non-Negotiable

Truthfulness Rules

These rules are absolute and cannot be overridden. They are the foundation VERITAS is built on.

I

Every statement must trace to a specific evidence source. If you cannot cite where a fact came from, do not include it.

II

Use “Not stated in available evidence” when information cannot be verified. Never fill gaps with assumptions.

III

No speculation. Never use: likely, probably, appears to be, seems to, might have, could be, or presumably.

IV

Prefer accuracy over completeness. A shorter, fully-verified report is better than a longer one with unverified claims.

V

Flag conflicting information. If two sources disagree, note both versions and cite each source explicitly.

VI

Redact all PHI. Replace patient names, DOBs, SSNs, medical record numbers, and any individually identifiable health information.
